# Bug: daemon install ignores configured port

## Bug Description\n\n\`clawdbot --profile jbphoenix daemon install\` does NOT respect the configured gateway port from \`~/.clawdbot-jbphoenix/clawdbot.json\`.\n\nInstead, it uses:\n- The default port (18789), OR\n- The port of an already running gateway instance\n\n### Expected Behavior\n\n\`daemon install\` should read the \`gateway.port\` setting from the profile's config file and use that port in the generated LaunchAgent plist.\n\n### Actual Behavior\n\nGenerated LaunchAgent plist contains wrong port (e.g., 18789 instead of configured 28800 for jbphoenix profile).\n\n### Reproduction\n\n```bash\n# Profile configured with port 28800\ncat ~/.clawdbot-jbphoenix/clawdbot.json | jq '.gateway.port'\n# Output: 28800\n\n# Install daemon\nclawdbot --profile jbphoenix daemon install\n\n# Generated plist has wrong port\ncat ~/Library/LaunchAgents/com.clawdbot.jbphoenix.plist | grep -A6 ProgramArguments\n# Output shows: --port 18789 (WRONG!)\n```\n\n### Affected Areas\n\n1. LaunchAgent plist ProgramArguments\n2. CLAWDBOT_GATEWAY_PORT environment variable\n\n### Suggested Fix\n\n\`daemon install\` should:\n1. Read \`gateway.port\` from the profile's config file\n2. Use that value in the generated plist's ProgramArguments\n3. Set CLAWDBOT_GATEWAY_PORT env var to match\n\n### Workaround\n\nManually edit the plist after installation to correct the port.\n\n---\nReported: 2026-01-16
